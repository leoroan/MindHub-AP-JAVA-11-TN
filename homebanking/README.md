# HOMEBANKING-TASK6 -- MindHub-AP-JAVA-11-TN

Repo con el TP 6 del BootCamp de MindHub-AP JAVA

## SEGUNDA PARTE DEL CURSO

Escribiré código para que los cliente puedan:

* Registrarse como nuevos clientes
* Iniciar sesión como clientes ya registrados
* Crear cuentas
* Realizar transferencias de dinero
* Solicitar tarjetas de crédito y débito
* Solicitar préstamos
* Mejorar la aplicación a nivel de código y funcionalidades
* El énfasis está ahora en implementar los servicios y algoritmos necesarios en Java para que se puedan realizar todas las operaciones.

## OBJETIVOS
1, Implementar la capa de seguridad de una aplicación Java utilizando el módulo Spring Security del framework Spring

2. Implementar algoritmos para brindar lógica de negocio a las operaciones disponibles en una API REST de un home banking

3. Desplegar una aplicación Java en servicios de computación en la nube como Heroku o Netify

4. Validar que el código generado sea correcto y siga los estándares de la industria.

## DESARROLLO DE ALGORITMOS
 - ¿Qué es el desarrollo de algoritmos?   Lo que hace que los programas de computadora sean interesantes y útiles son los algoritmos que implementan. Los algoritmos son procedimientos para tomar datos, calcular valores y tomar decisiones. El campo de la informática se centra en el desarrollo de algoritmos para hacer cosas que actualmente no se pueden hacer o que no se pueden hacer con la suficiente rapidez. Los algoritmos son cómo los motores de búsqueda pueden encontrar una página web para ti a partir de millones de páginas en una fracción de segundo, cómo se puede lanzar una sonda espacial desde la Tierra y encontrarse con un cometa a millones de millas de distancia, y así sucesivamente.  
 
En este curso, aprenderá a desarrollar e implementar en Java algunos algoritmos simples pero no triviales para:
Determinar si un cliente puede crear cuentas, préstamos y tarjetas.
Actualizar las cuentas de los clientes en base a las transferencias realizadas.
Calcular las tasas de interés de los préstamos.
Generar número de tarjetas y códigos CVV de manera automática.
Verificar que solo se puedan realizar operaciones sobre la cuentas del usuario autenticado.
